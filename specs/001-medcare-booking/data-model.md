## Entities

### 1. Patient (auth.users — managed by Supabase Auth)

Managed entirely by Supabase Auth. Not a custom table.

| Field | Type | Description |
|-------|------|-------------|
| `id` | UUID | Auto-generated by Supabase Auth. Used as FK in `appointments.user_id` |
| `email` | TEXT | Patient's email address. Used for login. |
| `raw_user_meta_data.full_name` | TEXT | Patient's display name. Set at registration. |
| `created_at` | TIMESTAMPTZ | Account creation time |

---

### 2. Doctor

Pre-seeded, read-only from the patient-facing app.

| Field | Type | Constraints | Description |
|-------|------|-------------|-------------|
| `id` | UUID | PK, auto-generated | Unique doctor identifier |
| `name` | TEXT | NOT NULL | Full name with title (e.g., "Dr. James Carter") |
| `specialty` | TEXT | NOT NULL, check constraint | One of 5 defined specialties |
| `bio` | TEXT | nullable | 1–2 sentence professional summary |
| `location` | TEXT | NOT NULL, check constraint | One of: 'New York', 'London', 'Dubai' |
| `avatar_color` | TEXT | NOT NULL | Hex color for initials avatar background |

**Relationships**: One doctor → many appointments

---

### 3. Appointment

| Field | Type | Constraints | Description |
|-------|------|-------------|-------------|
| `id` | UUID | PK, auto-generated | Unique appointment identifier |
| `user_id` | UUID | FK → auth.users(id), NOT NULL | The patient who booked |
| `doctor_id` | UUID | FK → doctors(id), NOT NULL | The booked doctor |
| `patient_name` | TEXT | NOT NULL | Patient's name (copied from auth metadata at booking time) |
| `patient_email` | TEXT | NOT NULL | Patient's email (copied from auth session at booking time) |
| `patient_phone` | TEXT | NOT NULL | Patient contact phone (entered in booking form) |
| `appointment_date` | DATE | NOT NULL | Selected date (Mon–Sat only) |
| `time_slot` | TEXT | NOT NULL | e.g., '10:00 AM' |
| `location` | TEXT | NOT NULL | Clinic location (copied from doctor at booking time) |
| `status` | TEXT | NOT NULL, default 'confirmed' | 'confirmed' or 'cancelled' |
| `created_at` | TIMESTAMPTZ | NOT NULL, default now() | Record creation time |

**Relationships**: Many appointments → one patient (via `user_id`), many appointments → one doctor

**State transitions**:
```
confirmed → cancelled   (patient or admin; irreversible in MVP)
```

**Validation rules**:
- `appointment_date` must not be a Sunday
- `appointment_date` must be within 30 days from today
- `time_slot` must be valid for the given day of week
- `status` only transitions confirmed → cancelled

## Supabase SQL Schema

```sql
-- Enable UUID extension
create extension if not exists "pgcrypto";

-- Doctors table (seeded manually)
create table public.doctors (
  id           uuid primary key default gen_random_uuid(),
  name         text not null,
  specialty    text not null check (specialty in ('Oncology','Pulmonology','Cardiology','Internal Medicine','Family Medicine')),
  bio          text,
  location     text not null check (location in ('New York','London','Dubai')),
  avatar_color text not null default '#0f3460'
);

-- Allow public (anon) reads on doctors
alter table public.doctors enable row level security;
create policy "public_read_doctors" on public.doctors
  for select using (true);

-- Appointments table
create table public.appointments (
  id               uuid primary key default gen_random_uuid(),
  user_id          uuid not null references auth.users(id) on delete cascade,
  doctor_id        uuid not null references public.doctors(id),
  patient_name     text not null,
  patient_email    text not null,
  patient_phone    text not null,
  appointment_date date not null,
  time_slot        text not null,
  location         text not null,
  status           text not null default 'confirmed' check (status in ('confirmed','cancelled')),
  created_at       timestamptz not null default now()
);

-- Row Level Security for appointments
alter table public.appointments enable row level security;

-- Patients can read their own appointments
create policy "patients_read_own" on public.appointments
  for select using (auth.uid() = user_id);

-- Patients can insert their own appointments
create policy "patients_insert_own" on public.appointments
  for insert with check (auth.uid() = user_id);

-- Patients can cancel (update) their own confirmed appointments
create policy "patients_cancel_own" on public.appointments
  for update using (auth.uid() = user_id and status = 'confirmed')
  with check (status = 'cancelled');

-- Index for patient lookups
create index idx_appointments_user_id on public.appointments (user_id);
-- Index for admin queries sorted by date
create index idx_appointments_date on public.appointments (appointment_date desc);
```

## TypeScript Interfaces (`lib/types.ts`)

```typescript
export interface Doctor {
  id: string;
  name: string;
  specialty: Specialty;
  bio: string | null;
  location: ClinicCity;
  avatar_color: string;
}

export interface Appointment {
  id: string;
  user_id: string;
  doctor_id: string;
  patient_name: string;
  patient_email: string;
  patient_phone: string;
  appointment_date: string;   // ISO date string YYYY-MM-DD
  time_slot: string;           // e.g. '10:00 AM'
  location: ClinicCity;
  status: AppointmentStatus;
  created_at: string;
  doctor?: Doctor;             // joined from query
}

export interface ClinicLocation {
  city: ClinicCity;
  address: string;
  phone: string;
  hours_weekday: string;
  hours_saturday: string;
  timezone: string;
}

export type Specialty = 'Oncology' | 'Pulmonology' | 'Cardiology' | 'Internal Medicine' | 'Family Medicine';
export type ClinicCity = 'New York' | 'London' | 'Dubai';
export type AppointmentStatus = 'confirmed' | 'cancelled';

export interface BookingFormData {
  doctor: Doctor | null;
  date: string | null;         // ISO date YYYY-MM-DD
  timeSlot: string | null;
  patientPhone: string;        // name + email come from auth session
}
```

## Admin Credentials (`lib/admin-auth.ts`)

```typescript
// Server-side only — never imported in client components
export function validateAdminCredentials(username: string, password: string): boolean {
  return (
    username === process.env.ADMIN_USERNAME &&
    password === process.env.ADMIN_PASSWORD
  );
}
```

**Environment variables** (`.env.local`):
```env
ADMIN_USERNAME=admin
ADMIN_PASSWORD=Admin1
```

> **Note**: Hardcoded for MVP only. Production requires hashed credentials and proper admin user management.
